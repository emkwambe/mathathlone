# MathAthlone — Geometry & Visual Question Technical Guide
### Ensuring Zero Skill Gaps from Technology Limitations

**Version:** 1.0 — June 2026
**For:** Content authors, curriculum designers, developers
**Mpingo Systems LLC**

---

## The Problem

A competition platform that claims to cover geometry but can only render text-based questions creates invisible skill gaps. Students practice algebraic manipulation because it works in a text input, but never practice spatial reasoning because the platform can't represent it. Over time, the platform silently trains students to be good at algebra and weak at geometry — the opposite of balanced assessment.

This guide catalogs every visual/spatial skill in the NC Math curriculum, classifies each by what technology it requires, and specifies exactly how MathAthlone must represent it. No skill gets skipped because of a rendering limitation.

---

## 1. Representation Taxonomy

Every visual math concept falls into one of five representation categories:

### Category A: Static SVG (fully solvable from a rendered image)
The student sees an SVG image and selects or types an answer. No interaction with the image needed. The SVG is the stimulus; the answer is text.

**Tech:** Inline SVG generated by visual-generators.ts. Already implemented for 12 generators.

**Example:** "What type of correlation does the scatter plot show?" → Student sees SVG scatter plot → picks "Positive correlation."

### Category B: Labeled Diagram (SVG with embedded measurements/labels)
The student reads values from a labeled diagram and computes an answer. The diagram provides the data; the math is done mentally or on scratch paper.

**Tech:** SVG with text labels for lengths, angles, coordinates. Student types a numeric answer. No interaction with the image.

**Example:** "Find the area of the triangle." → SVG shows triangle with base=8, height=5 labeled → Student types "20."

### Category C: Multi-Panel Comparison (multiple images, pick the correct one)
The student compares 2-4 images and identifies which one matches a criterion.

**Tech:** 2x2 grid of mini-SVGs, each labeled A/B/C/D. Student picks a letter. Already implemented for vertical_line_test generator.

**Example:** "Which shape has exactly 2 lines of symmetry?" → Four shapes rendered → Student picks "C."

### Category D: Construction/Manipulation (requires interaction with the image)
The student must draw, drag, rotate, or place objects on a coordinate plane. This is the hardest category — text/MC cannot assess it without losing the skill.

**Tech:** Interactive HTML canvas or React component. Student manipulates elements, system validates the result. NOT YET IMPLEMENTED.

**Example:** "Plot the point (3, -2) on the coordinate plane." → Student clicks on the grid → System checks if click is at (3, -2).

### Category E: Physical/Tactile (cannot be digitally assessed)
Skills that fundamentally require physical materials: compass-and-straightedge constructions, paper folding, physical measurement with rulers/protractors.

**Tech:** Cannot be assessed digitally without losing the core skill. Must be handled with alternative question framing that tests the *understanding* behind the construction rather than the motor skill.

**Example:** Instead of "Construct the perpendicular bisector of segment AB" → Ask "Point P is on the perpendicular bisector of AB. If A = (1, 3) and B = (7, 3), what is the x-coordinate of P?"

## 4. Solution Architecture

### 4.1 Immediate: Category B Generators (Labeled Diagrams)

These require no new UI components — just SVG generators that embed measurement labels.

**SVG Labeled Diagram Pattern:**
```typescript
function labeledTriangle(base: number, height: number, showArea: boolean): string {
  // Draw triangle
  // Add dimension lines with arrows
  // Add text labels: "8 cm", "5 cm"
  // Optionally shade the interior
  // Return SVG string
}
```

**New generators needed (Priority 1):**

| Generator | SVG Content | Question Frame | Answer Type |
|---|---|---|---|
| `read_slope_from_graph` | Coordinate plane with plotted line + 2 highlighted points | "What is the slope of the line shown?" | number_or_fraction |
| `estimate_scatter_value` | Scatter plot with trend line + vertical dotted line at target x | "Estimate y when x = 12" | number_or_fraction |
| `read_histogram` | Histogram with labeled bins + frequency axis | "How many data points are between 20 and 30?" | integer |
| `read_box_plot` | Box plot with labeled number line | "What is the IQR?" | number_or_fraction |
| `compare_box_plots` | Two side-by-side box plots | "Which group has a higher median?" | text (MC) |
| `triangle_angle_sum` | Triangle with 2 angles labeled | "Find the missing angle." | integer |
| `pythagorean_find_side` | Right triangle with 2 sides labeled | "Find the missing side." | number_or_fraction |
| `transversal_angles` | Parallel lines + transversal with some angles labeled | "Find angle x." | integer |
| `composite_area` | L-shape or combined rectangle with dimensions | "Find the total area." | number_or_fraction |
| `isometric_volume` | Rectangular prism with l, w, h labeled | "Find the volume." | integer |

**SVG Utility Functions Needed:**
```typescript
// Dimension line with arrows and label
function dimensionLine(x1, y1, x2, y2, label: string): string

// Angle arc with degree label
function angleArc(cx, cy, radius, startAngle, endAngle, label: string): string

// Number line for box plots
function numberLineWithBoxPlot(min, max, q1, median, q3, whiskerLow, whiskerHigh): string

// Histogram bars
function histogramBars(bins: {label: string, count: number}[]): string

// Isometric rectangular prism
function isometricBox(length, width, height, labels: boolean): string
```

### 4.2 Near-Term: Category C Generators (Multi-Panel Comparison)

These use the existing 2x2 mini-panel pattern from the vertical_line_test generator.

**New generators needed (Priority 2):**

| Generator | Panel Content | Question Frame |
|---|---|---|
| `which_graph_matches_equation` | 4 coordinate planes with different lines | "Which graph represents y = -2x + 3?" |
| `which_transformation_result` | Original figure + 4 candidate images | "Which shows △ABC after reflection across the y-axis?" |
| `congruent_or_similar` | 4 pairs of figures | "Which pair is congruent (not just similar)?" |
| `which_box_plot_matches` | 4 box plots | "Which box plot could represent the data set {2, 5, 7, 8, 9, 12, 15}?" |
| `which_histogram_matches` | 4 histograms | "Which histogram shows a right-skewed distribution?" |

### 4.3 Long-Term: Category D (Interactive Construction)

These require a new React component: `InteractiveGrid`.

**Architecture:**
```typescript
interface InteractiveGridProps {
  xMin: number; xMax: number;
  yMin: number; yMax: number;
  mode: 'plot_point' | 'draw_line' | 'draw_segment' | 'place_shape';
  existingElements?: SVGElement[];  // Pre-drawn content
  onSubmit: (result: GridResult) => void;
  timeLimit?: number;
}

interface GridResult {
  points?: {x: number, y: number}[];
  lines?: {slope: number, intercept: number}[];
  segments?: {x1: number, y1: number, x2: number, y2: number}[];
}
```

**Validation:**
```typescript
// "Plot the point (3, -2)"
function validatePointPlot(submitted: Point, target: Point, tolerance: number): boolean {
  return Math.abs(submitted.x - target.x) < tolerance 
      && Math.abs(submitted.y - target.y) < tolerance;
}

// "Graph y = 2x + 1"
function validateLinePlot(submitted: Line, target: Line): boolean {
  return Math.abs(submitted.slope - target.slope) < 0.1 
      && Math.abs(submitted.intercept - target.intercept) < 0.5;
}
```

**Implementation priority:** After Grade 7 + Grade 8 courses are built. The MC/FR workarounds (Category C reframes) cover 80% of the assessment need. Interactive grid covers the remaining 20% for students who need to demonstrate *production* skills, not just *recognition*.

### 4.4 Category E Reframes (Physical → Digital)

Skills that are inherently physical get reframed to test the underlying understanding:

| Physical Skill | Digital Reframe | What's Preserved | What's Lost |
|---|---|---|---|
| Compass-and-straightedge construction | "Which point is equidistant from A and B?" | Understanding of perpendicular bisector | Motor skill of using compass |
| Protractor measurement | "Angle ABC measures 55°. Classify it." | Angle classification knowledge | Reading a protractor |
| Ruler measurement | "The segment is 4.7 cm. Convert to mm." | Unit conversion | Physical measurement |
| Paper folding for symmetry | "How many lines of symmetry does this shape have?" | Symmetry understanding | Tactile exploration |
| Net folding for 3D shapes | "Which net folds into a cube?" (4 panel MC) | Spatial reasoning | Physical manipulation |

**Important:** The Pilot Test Guide should note which skills are reframed and why, so teachers understand the platform assesses *understanding* of these concepts, not the physical execution.

## 5. SVG Rendering Standards

### 5.1 Coordinate Planes

All coordinate plane SVGs must follow these standards:

```
Canvas:     360 × 280 px (matches existing visual-generators.ts)
Padding:    40px all sides
Grid:       Light gray (#e2e8f0), 0.5px stroke
Axes:       Dark slate (#334155), 1.5px stroke
Tick labels: Every integer (if range ≤ 10), every 2 (if range > 10)
Line colors: Indigo (#6366f1), Red (#ef4444), Green (#10b981), Amber (#f59e0b)
Point radius: 4px (data points), 5px (highlighted points)
Font:       System sans-serif, 10-13px
```

### 5.2 Labeled Diagrams

```
Dimension lines: 1px stroke, small arrowheads at endpoints
Labels:          13px bold, positioned at midpoint of dimension line
                 Offset 12px perpendicular to the line to avoid overlap
Angle arcs:      Radius 20-30px from vertex, 1.5px stroke
Angle labels:    11px, positioned at midpoint of arc, offset outward
Shading:         Fill with 15% opacity of the element color
```

### 5.3 Geometric Figures

```
Triangles:    Fill with 15% opacity blue, 2px stroke indigo
Rectangles:   Fill with 15% opacity green, 2px stroke green
Circles:      Fill with 15% opacity amber, 2px stroke amber
Right angle:  Small square (8x8px) at the vertex
Congruence:   Mark equal sides with tick marks (1, 2, or 3 ticks)
Parallel:     Mark with arrow chevrons on the lines
```

### 5.4 Statistical Graphics

```
Histograms:  Bar width = (plot width) / bins, 2px gap between bars
             Fill: indigo at 60% opacity, 1px indigo stroke
             x-axis: bin labels centered under bars
             y-axis: frequency labels at ticks

Box plots:   Box from Q1 to Q3, line at median
             Whiskers extend to min/max (or 1.5×IQR if teaching outliers)
             Box fill: indigo at 20% opacity
             Median line: 2px red stroke
             Number line below with labeled ticks

Scatter plots: 4px indigo dots, 15% opacity trend line (if shown)
              Axes labeled with variable names
              Grid lines at major intervals
```

### 5.5 3D Figures (Isometric Projection)

```
Isometric angle: 30° from horizontal (standard)
Front face:      Fill with 20% opacity blue
Top face:        Fill with 15% opacity blue (lighter = farther from viewer)
Side face:       Fill with 25% opacity blue (darker = shadow side)
Edge strokes:    1.5px dark slate
Hidden edges:    Dashed 1px gray (optional, for transparency)
Dimension labels: Along visible edges only
```

**Deliverable:** `src/components/competition/InteractiveGrid.tsx`

**Defer until:** After Grade 7 + Grade 8 courses are built and the MC/FR workarounds have been validated through pilot testing.

## 7. Quality Checklist for Visual Generators

Before any visual generator ships:

- [ ] SVG renders correctly at 360×280 on desktop AND mobile (< 400px width)
- [ ] All text labels are readable at the default viewBox size
- [ ] Dimension lines don't overlap with other elements
- [ ] Angle arcs don't extend outside the visible area
- [ ] Colors follow the standard palette (Section 5)
- [ ] Right angles marked with small squares (not arcs)
- [ ] Equal sides marked with tick marks
- [ ] Answer is not visually obvious (e.g., don't make the correct option a different color)
- [ ] Distractors are plausible (wrong answers should represent common mistakes)
- [ ] Generator tested with 20+ random seeds to verify no degenerate cases
- [ ] Mobile: touch targets for MC options are ≥ 44px (WCAG)
